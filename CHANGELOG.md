# Changelog

All notable changes to the TekMemo monorepo are documented here. Per-package
changelogs (generated by Changesets) live alongside each package; this file is
the human-readable, repo-level narrative and the single source of truth for
release dates.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

---

## [1.0.0-beta.0] — 2026-06-29

**Public beta.** The local-first runtime is feature-complete for its intended
use cases; the public API is stabilizing toward 1.0. The hosted TekMemo Cloud
layer remains in separate early access.

### Added
- **Intelligent local recall** — hybrid retrieval (BM25 + fuzzy token matching)
  works with zero configuration. Add an embedder adapter for semantic recall; add
  the local ONNX embedder (`@tekbreed/tekmemo-adapter-transformers`) for
  zero-API-key hybrid recall in-process.
- **Best-effort write path** — a failing or missing embedder never breaks a
  write; memory stays discoverable through lexical recall.
- **AI SDK runtime via `createAiSdkRuntimeFromTekmemo`** — every recall flows
  through the intelligent hybrid engine, never a naive text search. (See
  [ADR 0001](./docs/adr/0001-ai-sdk-runtime-via-tekmemo.md).)
- **Agent-context directive** — `buildContext` emits a directive preamble so all
  runtimes (local/cloud/hybrid) instruct the agent consistently.
- **`tekmemo generate agent-rules`** — emits a ≤50-line agent-instructions file
  for Codex, Claude Code, Gemini CLI, Copilot, and Cursor, each with its
  platform-correct MCP config pointer.
- **MCP server** — stdio + HTTP transports; agent-directing tool descriptions;
  local-embedder wiring (zero-config hybrid recall).

### Changed
- Repository renamed to **TekMemo** in all user-facing prose. Org/legal entity
  remains TekBreed (see the LICENSE copyright line).
- Tooling packages renamed to match their directory names
  (`@repo/tsdown`, `@repo/typescript`, `@repo/utils`); `@repo/test-utils` removed.

### Removed
- Legacy naive AI SDK runtime (`createLocalAiSdkRuntime` + `local-runtime.ts`) —
  superseded by the Tekmemo-delegating runtime.

## [1.0.0-alpha.0] — 2026-06-15

**Initial public alpha** of the rewritten monorepo. Published to npm as
`@tekbreed/tekmemo`, `@tekbreed/tekmemo-cli`, and `@tekbreed/tekmemo-mcp-server`.

### Added
- **Core memory runtime** — the `Tekmemo` client; `core`, `notes`, `recent`, and
  `graph` memory; file-first stores under `.tekmemo/`.
- **Four runtime modes** — `local` (filesystem), `cloud` (TekMemo Cloud API),
  `hybrid` (local + cloud with read/write policies), `memory` (volatile, for tests).
- **Cloud sync** — client-side sync against TekMemo Cloud.
- **AI SDK integration** — memory tools, runtime, and agent-session helpers.
- **Embedder adapters** — OpenAI, Voyage (embedder + reranker), Upstash (vector
  recall store), Transformers (local ONNX).
- **Reranking** — with deterministic fallback when a reranker is unavailable.
- **AgentFS** — agent-session persistence.
- **Benchmark kit** — workloads and runners for memory evaluation
  (`@tekbreed/tekmemo-benchmark-kit`).
- **Testing utilities** — shared contract tests, fakes, and fixtures
  (`@tekbreed/tekmemo-testing`).
