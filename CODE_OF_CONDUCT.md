# Code of Conduct

## Our commitment

TekMemo is an open-source project for developers building memory infrastructure for AI apps and agents.

We want this project to be a respectful, practical, and productive environment for everyone.

We are committed to providing a space that is welcoming regardless of:

- level of experience
- education
- nationality
- background
- language
- personal identity
- technical preference
- location
- company affiliation

The goal is simple:

> People should be able to contribute, ask questions, disagree, and learn without being harassed or disrespected.

---

## Expected behavior

Participants are expected to:

- be respectful
- be constructive
- assume good intent when reasonable
- ask clear questions
- give actionable feedback
- stay focused on the work
- explain technical disagreements clearly
- accept maintainer decisions gracefully
- help newcomers where possible
- avoid personal attacks
- respect project boundaries and contributor time

Good examples:

```txt
"I think this API may be difficult to use because..."
"Can we add a test for this edge case?"
"This package may be taking on too many responsibilities."
"I disagree with this approach; here is an alternative."
"Thanks for the PR. Could you add docs before we merge?"
```

---

## Unacceptable behavior

Unacceptable behavior includes:

* harassment
* threats
* personal insults
* discriminatory comments
* sexualized language or imagery
* trolling
* repeated disruption
* doxxing
* publishing private communication without consent
* spamming maintainers
* intentionally derailing technical discussions
* abusive comments in issues, PRs, discussions, or community spaces
* pressuring maintainers to merge, release, or respond immediately

This also includes behavior outside project spaces when it directly affects the safety or participation of project members.

---

## Technical disagreement

Technical disagreement is welcome.

TekMemo deals with complex topics:

* memory architecture
* AI provider adapters
* vector recall
* graph memory
* cloud sync
* MCP
* security
* observability
* package boundaries

Disagreement should focus on:

* correctness
* maintainability
* performance
* security
* usability
* package responsibility
* user impact

Disagreement should not become personal.

Bad:

```txt
"This is stupid."
"You clearly don't understand TypeScript."
"Only an amateur would design it this way."
```

Better:

```txt
"This design may create a package boundary issue because..."
"This API may be difficult to evolve. Could we consider..."
"This test does not cover the failure mode where..."
```

---

## Maintainer responsibilities

Maintainers are responsible for:

* clarifying standards
* reviewing reports
* enforcing this Code of Conduct
* keeping discussions productive
* making final decisions when needed
* protecting contributors from harassment
* removing inappropriate content when necessary

Maintainers may take action when behavior harms the project or community.

---

## Enforcement actions

Depending on the situation, maintainers may:

* remind a participant of the Code of Conduct
* edit or remove comments
* lock discussions
* close issues or pull requests
* ask a participant to stop specific behavior
* temporarily restrict participation
* permanently ban a participant from project spaces
* report serious abuse to the relevant platform

Enforcement decisions are based on impact, severity, and repeated behavior.

---

## Reporting

If you experience or witness unacceptable behavior, report it to:

```txt
conduct@tekmemo.dev
```

If that email is not yet active, contact a maintainer privately through GitHub.

Please include:

* what happened
* where it happened
* when it happened
* who was involved
* links or screenshots if available
* whether there is immediate risk

Reports will be handled as privately as reasonably possible.

---

## False or bad-faith reports

Reports made in good faith are welcome, even if they are difficult to verify.

Reports made intentionally to harass, retaliate, or mislead may themselves be considered a violation of this Code of Conduct.

---

## Scope

This Code of Conduct applies in:

* GitHub issues
* pull requests
* discussions
* documentation spaces
* community chats
* official project events
* maintainer interactions
* any official TekMemo OSS community space

It may also apply outside official spaces when behavior directly affects the project or community.

---

## Contributor expectations

Contributors are not expected to be perfect.

People can make mistakes.

When corrected, participants are expected to listen, adjust, and move forward.

Repeated harmful behavior after correction is not acceptable.

---

## Maintainer availability

TekMemo may be maintained by a small team.

Please respect that maintainers may not respond immediately.

Do not repeatedly ping maintainers for:

* issue triage
* PR review
* release timing
* roadmap decisions
* support requests

A clear, respectful follow-up is acceptable.

Repeated pressure is not.

---

## Project-first culture

The TekMemo community should optimize for:

* useful software
* clear documentation
* safe defaults
* maintainable APIs
* thoughtful architecture
* beginner-friendly explanations
* production-grade engineering

The best contribution is one that makes the project easier to use, safer to run, or easier to maintain.

---

## Acknowledgment

By participating in TekMemo spaces, you agree to follow this Code of Conduct.
