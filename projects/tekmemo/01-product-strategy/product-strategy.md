# TekMemo Product Strategy

## 1. Product thesis

TekMemo is a **file-first memory runtime for AI apps and agents**.

It gives developers a way to make AI memory:

- durable
- inspectable
- editable
- portable
- retrieval-ready
- provider-adapter-friendly

TekMemo Cloud is the hosted layer for teams that do not want to operate the memory infrastructure themselves.

---

# 2. What TekMemo is

TekMemo is a memory system for AI products.

It provides:

- memory files
- memory stores
- memory records
- semantic recall
- reranking
- provider adapters
- benchmark tools
- cloud API and dashboard

The key idea is that memory should not be hidden inside a black-box vector database.

It should be visible, versionable, inspectable, and retrievable.

---

# 3. What TekMemo is not

TekMemo is not only:

- a vector database
- a RAG framework
- chat history
- a notes app
- an AI chatbot

TekMemo sits below those products as memory infrastructure.

---

# 4. Core positioning

## One-line positioning

**File-first memory runtime and hosted memory infrastructure for AI apps and agents.**

## Developer-facing promise

Build AI apps that remember without locking memory inside a black box.

## Cloud-facing promise

Use hosted memory, recall, API keys, usage limits, and project dashboards without building the backend yourself.

# 6. Core use cases

## Agent personalization

Store persistent preferences and working assumptions so an agent can improve over time.

## Project memory

Remember technical decisions, constraints, product direction, repo notes, and workflow rules for a project.

## Coding-agent memory

Keep repo-specific context available across sessions.

## Team memory

Share project memory across team members and AI workflows.

## Memory inspection and audit

Let humans inspect, edit, delete, and correct memory.

## Local-first development

Use memory locally before adopting cloud sync.

---

# 7. Strategic wedge

Many memory tools compete on retrieval only.

TekMemo should compete on:

- file-first memory
- developer ownership
- inspectability
- BYOK
- OSS adapters
- cloud convenience
- benchmarkable quality
- agent workflow integration

---

# 8. Product pillars

## Pillar 1 — Own your memory

Memory should be exportable and inspectable.

## Pillar 2 — Start free locally

Developers should be able to test the memory runtime without paying.

## Pillar 3 — Sync when useful

Cloud should be useful when teams need hosted sync, API keys, dashboards, and billing.

## Pillar 4 — Debug memory like code

Memory should have records, sources, versions, and provenance.

## Pillar 5 — Beat on workflow

TekMemo should not be only better search. It should make agent memory workflows easier.
