# TekMemo Competitive Positioning

## 1. Competitor categories

TekMemo competes near several categories:

| Category | Examples | TekMemo position |
|---|---|---|
| Memory APIs | Mem0, Supermemory | More inspectable, file-first, OSS-friendly |
| Agent runtimes | Letta | Lighter memory runtime and adapter stack |
| Vector DBs | Upstash, Pinecone, Qdrant | TekMemo orchestrates memory around vector stores |
| RAG frameworks | LlamaIndex, LangChain memory utilities | More productized memory file/runtime layer |
| Notes/knowledge tools | Notion-like memory apps | Developer-first, API-first, agent-first |

---

# 2. Differentiation

TekMemo should not claim to be the biggest model memory platform.

It should claim:

- memory you can inspect
- memory you can edit
- memory you can move
- memory you can benchmark
- memory you can run locally
- memory you can host when ready

---

# 3. Competitive wedge

The wedge is:

```txt
Open-source file-first memory runtime + hosted cloud for teams that want convenience.
```

This lets TekMemo appeal to developers before asking them to trust the cloud product.
